# design-sync notes — IndieWork

Repo is the **IndieWork Next.js app** (Next 16, React 19, Tailwind v4), NOT a packaged
component library. There is **no library `dist/` and no build that emits an ESM entry**
(`next build` → `.next/`, not consumable). So this is **synth-entry mode**: the converter
bundles directly from `src/`.

## Source shape
- Design-system surface = presentational primitives in `src/components/ui/`.
- Styling: **semantic CSS classes + inline CSS-var styles** (NOT Tailwind utilities).
  The scoped components use no Tailwind utility classes.
- Stylesheets (plain hand-written CSS, no `@apply`/`@theme`/`@tailwind`):
  - `src/styles/tokens.css` — design tokens (colors, radii, shadows, status/priority palettes), `.chip`/`.dot`.
  - `src/styles/app.css` — app-shell + component styles.
  - `src/styles/screens.css` — the bulk of component styles (`.meta-tag`, `.st-chip`, `.pri-bars`, `.progress`, `.due-pill`, `.circle-check`, `.ref-tag`, `.popover`, `.opt`, `.modal*`, `.login-wordmark` …).
  - `src/app/globals.css` is just `@import 'tailwindcss'` + maps `--font-ui`→`var(--font-hanken)`. Tailwind otherwise unused by the scoped components.
- Path alias: `@/* → ./src/*` (tsconfig.json `compilerOptions.paths`). Set `cfg.tsconfig`.
- `@/lib/domain` and `@/lib/dates` are explicitly **client-safe** (no server/db imports) — safe to bundle.

## Scoped IN (clean, bundlable, presentational)
brand: BrandMark, Wordmark · bits: StatusChip, PriorityBars, ModuleTag, MilestoneTag, DuePill, Progress, EntityIcon ·
interactive: CircleCheck, RefTag · modal: Modal · popover: Popover, OptionList · icon-picker: IconPicker · dyn-icon: DynamicLucide

## Scoped OUT (first sync)
- `ModuleIcon` — alias of `EntityIcon` (dedupe).
- `Ic` (object), `iconByName`, `isEmojiValue` — not components.
- Markdown/comment editors (`MarkdownEditor`, `MarkdownView`, `CommentComposer`, `CommentEditor` + `*Impl`):
  **`next/dynamic`-coupled + Tiptap-heavy** → won't bundle/render standalone. Add in a later re-sync if wanted.

## Fonts — SHIPPED (resolved as of the author-everything run)
- Brand UI font = **Hanken Grotesk** (`--font-ui`), mono = **IBM Plex Mono** (`--font-mono`, used by RefTag).
  In the app both come from `next/font/google` (obfuscated `--font-hanken`), which does NOT exist in the
  standalone bundle. So design-sync ships its own self-hosted woff2:
  - `.design-sync/fonts/hanken.css` + 3 woff2 (latin, latin-ext, vietnamese; variable 100–900) — matches `layout.tsx` subsets.
  - `.design-sync/fonts/ibm-plex-mono.css` + 2 woff2 (latin, weights 400/500).
  - Both wired via `cfg.extraFonts`; OFL-licensed; fetched from Google Fonts (Hanken v12, Plex Mono v20).
  - `build-css.mjs` binds `--font-ui` to `'Hanken Grotesk', …` at the end of `ds-combined.css` (the app's
    globals.css owns that slot via next/font, which the bundle lacks).
  Result: `[FONT_MISSING]` is gone; cards render in the real faces (RefTag confirmed monospace).

## CSS entry — DERIVED, regenerate before every build
- `cfg.cssEntry` = `.design-sync/ds-combined.css` is **gitignored + derived**: a concat of
  `src/styles/{tokens,app,screens}.css` + the `--font-ui` font-binding. It is regenerated by
  `cfg.buildCmd` = `node .design-sync/build-css.mjs`. **Run that before any design-sync build** (and on a
  fresh clone — the file won't exist). The build APPENDS cssEntry verbatim into `_ds_bundle.css` (it does
  NOT inline `@import`s), so a wrapper-with-@imports would break — keep it a real concat.

## Previews — all 16 authored (author-everything run)
- `.design-sync/previews/<Name>.tsx`, one per component, real JSX importing from `'indiework'` (+ real enums
  from `@/lib/domain` for sweeps). Each PascalCase export = one graded cell; all cells graded `good`.
- **Overlays** (click-gated or portal) use `cfg.overrides.<Name>` `{cardMode:"single", primaryStory, viewport}`:
  - `Modal` (always-open portal) → WorkspaceForm / ConfirmDelete, 720x560.
  - `IconPicker` + `Popover` (click-gated) → an `AutoOpen` effect that clicks `.pop-trigger` on mount opens
    the real popover for the card. primaryStory OpenOnIcons / StatusMenu; viewports 380x470 / 340x440.
  - Gotcha: adding/changing an override AFTER a full build makes `preview-rebuild` fail `[CONFIG_STALE]`
    for that component — re-run `package-build.mjs` (full) to re-stamp the manifest before capturing it.

## Re-sync risks (what the next run must watch)
- **`ds-combined.css` is gitignored + derived.** Always `node .design-sync/build-css.mjs` (cfg.buildCmd)
  before the build, else stale/missing CSS ships. On the next run the driver should rebuild it anyway.
- **Fonts are pinned to Google Fonts versions** (Hanken v12, Plex Mono v20) snapshotted as woff2 in
  `.design-sync/fonts/`. They're committed, so re-syncs are stable; only re-fetch if Google bumps a version
  AND you want it. If a font file goes missing, validate prints `[FONT_MISSING]`/`[FONT_DANGLING]`.
- **Date-relative previews** (DuePill, others using `Date.now()`) render states relative to the real clock,
  so the exact dates shown drift run-to-run — that's expected, not a regression. The overdue/soon/later
  *states* are what matter and stay correct.
- **Overlay previews depend on the `.pop-trigger` auto-open hack** (IconPicker, Popover). If lucide/popover
  internals rename that class upstream, those two cards collapse to just the trigger — re-grade them.
- **Scoped OUT still:** the Tiptap/next-dynamic markdown & comment editors are deliberately excluded
  (won't bundle standalone). Add them only with a real provider/bundling strategy.
